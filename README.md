# AWS Cleanup Tagged Resources

A dependency-aware Python cleanup tool to delete AWS resources for a specific stage/tag using a pre-generated inventory JSON file.

This project is designed to safely clean ephemeral environments (for example: `staging`, `coffee`) by:

- Reading resources from an inventory JSON generated by [`aws-list-resources`](https://github.com/welldone-cloud/aws-list-resources)
- Filtering resources by tag value (`Stage=staging`, `environment=staging`, etc.)
- Optionally matching by resource names when tags are missing
- Deleting resources in dependency-aware order
- Handling retry rounds for eventual consistency and in-use dependencies
- Supporting S3 versioned bucket deletion (versions, delete markers, multipart uploads, then bucket)

## Features

- Dry-run by default
- Explicit `--apply` for real deletion
- Strong confirmation prompt (`SURE`)
- Profile safety guard (`-dev` profiles by default)
- Include/exclude AWS resource types
- Multi-round deletion retries for dependency errors
- Generic fallback for unmatched resource types via AWS Cloud Control API (enabled by default)

Name matching safety:
- In `tag-or-name` mode, the script checks only identity-like fields per resource
  type (for example `DBInstanceIdentifier` for RDS), not every scalar property.
- This avoids false positives like matching a DB from an unrelated IAM role ARN.

## Requirements

- Python 3.9+
- AWS credentials configured locally (`aws configure` / profiles)
- IAM permissions to delete target resources

Install dependencies:

```bash
pip install -r requirements.txt
```

## Input Format

The script expects an inventory file with a structure similar to:

```json
{
  "_metadata": {
    "account_id": "123456789012"
  },
  "regions": {
    "us-east-1": {
      "AWS::S3::Bucket": {
        "my-bucket": {
          "BucketName": "my-bucket",
          "Tags": [{"Key": "Stage", "Value": "staging"}]
        }
      }
    }
  }
}
```

## Usage

### 1) Dry-run (recommended first)

```bash
python3 aws-cleanup-tagged-resources.py \
  --input /path/to/resources_account.json \
  --profile catalytic-pc-dev \
  --region us-east-1 \
  --tag-value staging
```

### 2) Apply deletion

```bash
python3 aws-cleanup-tagged-resources.py \
  --input /path/to/resources_account.json \
  --profile catalytic-pc-dev \
  --region us-east-1 \
  --tag-value staging \
  --apply
```

### 3) Tag-only matching

Use this mode when you want to delete only resources with explicit matching tags:

```bash
python3 aws-cleanup-tagged-resources.py \
  --input /path/to/resources_account.json \
  --profile catalytic-pc-dev \
  --region us-east-1 \
  --tag-value staging \
  --match-mode tag
```

### 4) Narrow to selected resource types

```bash
python3 aws-cleanup-tagged-resources.py \
  --input /path/to/resources_account.json \
  --profile catalytic-pc-dev \
  --region us-east-1 \
  --tag-value staging \
  --include-types AWS::ECS::Service,AWS::ECS::Cluster,AWS::S3::Bucket
```

## Main Options

- `--input`: inventory JSON path (required)
- `--tag-value`: stage/tag value to match (required)
- `--profile`: AWS profile (default: `catalytic-pc-dev`)
- `--region`: default AWS region (default: `us-east-1`)
- `--match-mode`: `tag` or `tag-or-name` (default: `tag-or-name`)
- `--include-types`: comma-separated allow list of AWS resource types
- `--exclude-types`: comma-separated block list of AWS resource types
- `--apply`: execute deletions (otherwise dry-run)
- `--force`: skip interactive `SURE` prompt (only with `--apply`)
- `--allow-non-dev-profile`: disable `-dev` profile guard
- `--max-rounds`: retry rounds for deferred deletions (default: `4`)
- `--round-wait-seconds`: delay between rounds (default: `20`)
- `--disable-generic-fallback`: disable generic delete attempts for resource types without dedicated handlers
- `--cloudcontrol-progress-seconds`: print Cloud Control polling progress every N seconds (default: `15`, set `0` to disable)

## Safety Notes

- Always run dry-run first.
- Start with narrow `--include-types` when testing.
- Use least-privilege credentials and explicit profiles.
- Deletions in AWS can be asynchronous. Re-run if needed.
- By default, matched resource types without a dedicated handler are attempted with AWS Cloud Control API so they are not silently excluded.
- Some AWS resource types are inherently non-deletable or require service-specific teardown; those will be reported as failed/deferred rather than skipped.

## Supported Resource Types

Current handlers include:

- `AWS::ECS::Service`
- `AWS::ECS::TaskDefinition`
- `AWS::AutoScaling::AutoScalingGroup`
- `AWS::EC2::Instance`
- `AWS::Lambda::Function`
- `AWS::Logs::LogGroup`
- `AWS::SQS::Queue`
- `AWS::SNS::Topic`
- `AWS::ElasticLoadBalancingV2::LoadBalancer`
- `AWS::ElasticLoadBalancingV2::TargetGroup`
- `AWS::ECS::Cluster`
- `AWS::EKS::Nodegroup`
- `AWS::EKS::FargateProfile`
- `AWS::EKS::Cluster`
- `AWS::ElastiCache::ReplicationGroup`
- `AWS::ElastiCache::CacheCluster`
- `AWS::RDS::DBInstance`
- `AWS::ECR::Repository`
- `AWS::OpenSearchService::Domain`
- `AWS::Elasticsearch::Domain`
- `AWS::S3::Bucket`

Unsupported matched resource types are listed for manual cleanup.
